⚙ Foundational SolidWorks Work

Mechanical Rotary Device

SolidWorks Mechanical Design Geneva Wheel Dog Clutch Parametric Modeling

This project involved the design and development of a novel dual-mode rotary mechanism that integrates two distinct power-transmission principles into a single compact assembly. Acting as a critical milestone in my technical development, this deep mechanical implementation serves as the foundational major work through which I built my practical command of SolidWorks and mechanical modeling pipelines.

Core Mechanism: The device combines a Geneva wheel (an intermittent motion mechanism commonly used in film transport and indexing machinery) with a dog clutch (a positive engagement coupling for direct torque transfer). The configuration allows the assembly to adaptively alternate output motion between discrete indexing steps and continuous, high-efficiency rotation from a unified input shaft.

  • Designed entirely from scratch in SolidWorks, executing full 3D parametric part configuration, detailed mating scripts, and interference checking.
  • Utilized to master advanced CAD features, including motion studies, clearance analysis, and structural layout configurations of complex physical interactions.
  • Engineered with a strict emphasis on fabricability — optimizing tolerances so components can be machined via traditional milling operations.
  • Developed a highly versatile mechanism with scalability for sorting matrices and custom packaging systems.
💡 Major Development Build: This engineering project was the catalyst for transitioning my CAD abilities to strong SolidWorks competency.
02

Grip Stability & Stress Analyzer

Arduino Uno R3 HX711 Load Cell Signal Processing Biomedical Sensing

A non-invasive physiological measurement system designed to estimate a user's stress level by analyzing grip force patterns and micro-tremors. The system leverages the relationship between psychological stress and involuntary muscle behavior — providing a quantitative, objective measurement tool for stress evaluation without clinical-grade equipment.

Hardware Stack: A calibrated load cell was interfaced with the HX711 24-bit ADC module, providing high-resolution force readings with minimal noise. The HX711 communicates with the Arduino Uno via a custom 2-wire protocol (CLK + DOUT). Continuous sampling at high frequency enables tremor frequency analysis in the time domain.

  • Load cell calibration performed using known reference weights to ensure measurement accuracy and repeatability.
  • HX711 configured for 80 SPS (samples per second) mode to capture rapid force fluctuations indicative of tremor.
  • Grip stability score computed from variance and standard deviation of the force signal over a defined sampling window.
  • Stress-level estimation derived by correlating measured tremor frequency and grip inconsistency with established physiological stress markers.
  • Serial output to PC for real-time visualization and logging of grip data over time.
💡 Demonstrates the application of embedded sensing for health tech — bridging hardware engineering and biomedical insight.
03

Adaptive Self-Mutating Authentication System

8051 (AT89S52) Embedded C Hardware Timers UART / Keypad Cybersecurity Workflow

A hardware-level digital authentication system designed on the 8051 (AT89S52) microcontroller that generates a fresh, non-deterministic password after every successful login event. The system directly addresses two common attack vectors on static PIN systems: replay attacks (capturing and reusing valid credentials) and shoulder-surfing (visual observation of PIN entry).

Authentication Logic: Hardware timers within the 8051 are exploited as a source of temporal entropy. Since timer values at the moment of login are highly variable (dependent on exact user interaction timing), they serve as a seed for a lightweight password mutation algorithm implemented in Embedded C. This ensures each generated password is statistically unique.

  • Password mutation algorithm implemented entirely in Embedded C — no external RNG module required.
  • 4×4 matrix keypad used for user input; 16×2 LCD for real-time feedback and prompt display.
  • Post-authentication, the new password is generated and displayed briefly, then stored in internal EEPROM/RAM for next-cycle validation.
  • Failed login attempt counter implemented — system locks after N incorrect tries to prevent brute-force entry.
  • Focuses on exploring operational reverse engineering and hardware execution workflows rather than abstract cryptographic code.
🔐 A firmware architecture project that bridges embedded structures and structural security workflows directly relevant to IoT system parameters.
04

Single-Axis Gimbal System

Arduino Nano MPU-6050 SG90 Servo PID Control Closed-Loop Feedback I2C Protocol

A real-time camera/payload stabilization system engineered to counteract angular disturbances on a single axis (pitch or roll). The system continuously reads inertial data from a 6-DOF IMU, computes the correction angle, and drives a micro servo to maintain a leveled platform — even as the base structure is tilted or disturbed.

Control Architecture: The MPU-6050 IMU communicates over I2C with the Arduino Nano, delivering accelerometer and gyroscope data at high sample rates. A complementary filter (weighted blend of accelerometer angle and integrated gyroscope rate) is applied to fuse sensor data and produce a stable, drift-corrected angle estimate.

  • MPU-6050 initialized over I2C at 0x68 address; raw 16-bit register values converted to physical units (°/s, m/s²).
  • Complementary filter implemented: angle = 0.98 × (angle + gyro × dt) + 0.02 × accel_angle — balancing gyro precision with accelerometer drift correction.
  • SG90 servo controlled via Arduino's Servo library, mapped from computed correction angle to PWM pulse width (1ms–2ms).
  • Loop frequency tuned to achieve sub-20ms response time, ensuring smooth and visually stable correction under dynamic disturbance.
  • Extensible to dual-axis (pitch + roll) by adding a second servo and expanding the filter to a full complementary or Kalman filter implementation.
⚙️ A clean demonstration of real-time closed-loop feedback control — the foundation of advanced robotics, drone stabilization, and autonomous systems.
